# Vulnerable Demo App

Purpose: build one deliberately vulnerable AI-enabled SaaS application that can be audited, exploited in a controlled environment, repaired, and verified.

## Required specification

- stack
- product concept
- user roles
- database structure
- payment flow
- AI feature
- intentional vulnerabilities
- expected secure behavior
- reproduction steps
- testing and fix-verification criteria

Do not add production credentials or real user data.
