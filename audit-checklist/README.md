# Audit Checklist

This directory will contain the repeatable review methodology derived from verified demo-app findings and later client evidence.

Each check should eventually define:

- objective
- affected component
- preconditions
- inspection procedure
- reproduction or test method
- expected secure behavior
- severity guidance
- common false positives
- evidence required before reporting
- remediation verification

Do not add a check solely because an AI model suggested it. Require evidence and human verification.
